fn splitting_evenly_in_with_min_size( &self, max_slices: usize, min_size: usize, ) -> SplitEvenlyIterator<'_, Self::Whole> ⓘ

Returns an iterator that iterates through evenly sized consecutive at max max_slices slices of self with every slice being atleast of size min_size.

§Precondition
  • max_slices > 0,
§Postcondition
  • If splitting exactly evenly is not possible, then slices on start would have bigger size than slices at end, still maintaining as even splitting as possible.
§Complexity
  • O(1) for RandomAccessCollection; otherwise O(n) where n == self.count().
§Example
use stl::*;

let arr = [1, 2, 3, 4, 5, 6, 7];
let splits: Vec<Vec<_>> = arr
    .splitting_evenly_in_with_min_size(3, 2)
    .map(|s| s.iter().copied().collect())
    .collect();
assert_eq!(splits, vec![vec![1, 2, 3], vec![4, 5], vec![6, 7]]);

fn partition_point<F>(&self, belongs_in_second_half: F) -> Self::Position
where F: FnMut(&Self::Element) -> bool,

Returns position of first element of collection for which predicate returns false.

§Precondition
  • The collection should be already partitioned wrt predicate i.e, there exist a position i such that predicate is true for every element of self.prefix_upto(i) and predicate is false for every element of self.suffix_from(i).
§Complexity
  • O(log n) for RandomAccessCollection, O(n) otherwise; where n == self.count().
  • O(log n) application of belongs_in_second_half.
§Example
use stl::*;

let arr = [1, 3, 5, 2, 4];
let i = arr.partition_point(|x| x % 2 == 0);
assert_eq!(i, 3);

fn partitioned<F>( &self, belongs_in_second_half: F, ) -> (Vec<Self::Element>, Vec<Self::Element>)
where Self::Element: Clone, F: FnMut(&Self::Element) -> bool,

Returns two Vec containing the elements of the collection that don’t and do satisfy the given predicate, respectively.

§Postcondition
  • Returns (falseVec, trueVec) where falseVec contains all elements that don’t satisfy predicate and trueVec contains all elements that do satisfy predicate.
  • Relative ordering of elements is preserved in both Vec.
§Complexity
  • O(n) where n == self.count().
§Examples
use stl::*;

let arr = [1, 2, 3, 4, 5];
let (evens, odds) = arr.partitioned(|x| x % 2 == 1);
assert_eq!(evens, [2, 4]);
assert_eq!(odds, [1, 3, 5]);

fn fold_left<R, F>(&self, init: R, op: F) -> R
where F: FnMut(R, &Self::Element) -> R,

Returns the result of combining elements of given collection using given accumulation operation from left to right.

§Postcondition
  • Result is (((init + e1) + e2) + ... + en). where e1, e2, …, en are the references to collection elements, where (a + b) represents op(a, b).
§Complexity:
  • O(count)
§Examples
use stl::*;

let arr = [1, 2, 3];
assert_eq!(arr.fold_left(0, |x, y| x + y), 6);
